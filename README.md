# YM
# 🎵 Yandex Music Track Collector

> Chrome-расширение для сбора треков из плейлистов Яндекс.Музыки.  
> Яндекс не даёт выгрузить плейлист — это расширение делает это за тебя.

---

## 🔥 Возможности

- ✅ Собирает треки **в порядке прокрутки** (как на странице)
- ✅ Работает с **виртуализированными списками** (треки загружаются при скролле)
- ✅ **Старт / Стоп** — парсинг только когда нужно
- ✅ **Бейдж на иконке** — видно количество треков без открытия попапа
- ✅ Экспорт в **TXT** и **CSV** (корректная кодировка, открывается в Excel)
- ✅ **Сброс** — очистить список и начать заново
- ✅ Убирает невидимые символы и неразрывные пробелы из названий

---

## 📦 Установка

> Расширение не опубликовано в Chrome Web Store — устанавливается вручную.

1. Скачай репозиторий:
   ```
   Code → Download ZIP
   ```

2. Открой Chrome и перейди по адресу:
   ```
   chrome://extensions/
   ```

3. Включи **«Режим разработчика»** (правый верхний угол)

4. Нажми **«Загрузить распакованное»** и выбери папку с расширением

5. Готово — иконка появится на панели браузера

## 🚀 Как использовать

### Шаг 1 — Откройте плейлист
Перейди на `music.yandex.ru` и открой нужный плейлист (свой или чужой).

### Шаг 2 — Запустите сбор
Кликни на иконку расширения → нажми кнопку **«Начать сбор»** (станет красной).

### Шаг 3 — Прокрутите список
Медленно прокручивай плейлист вниз. Треки будут появляться в списке по порядку.  
На иконке расширения виден счётчик собранных треков в реальном времени.

### Шаг 4 — Остановите и скачайте
Нажми **«Остановить сбор»**, затем скачай список:

| Кнопка | Формат | Содержимое |
|--------|--------|------------|
| ⬇ TXT | `.txt` | `Исполнитель — Название` |
| ⬇ CSV | `.csv` | `Artist,Title` (совместим с Excel) |

### Сброс
Кнопка **↺ Сброс** удаляет все собранные треки из памяти — можно начать заново.

## 🛠 Технические детали

- **Manifest Version 3** (актуальный стандарт Chrome)
- Для поиска треков в DOM используются **устойчивые селекторы** вида `[class*="Meta_title"]` — не ломаются при обновлении сайта
- **MutationObserver** отслеживает изменения DOM при виртуальной прокрутке
- Треки хранятся в **упорядоченном массиве** в `chrome.storage.local`
- Текст треков очищается от `\u00A0` (неразрывный пробел) и других невидимых символов

## ❓ Частые проблемы

**Расширение не видит треки**  
→ Нажми F5 на вкладке с Яндекс.Музыкой после установки, затем перезагрузи расширение в `chrome://extensions/`

**Видно предупреждение «Перезагрузите вкладку ЯМ»**  
→ Просто нажми F5 на вкладке с ЯМ. Content script загрузится заново.

**Треки не добавляются при скролле**  
→ Убедись, что кнопка «Начать сбор» нажата (красная). Без этого парсинг не идёт.

**CSV открывается с кракозябрами в Excel**  
→ Файл сохраняется с BOM (`UTF-8 with BOM`) — Excel должен открывать корректно. Если нет: Data → From Text/CSV → выбери UTF-8.

---

## 📌 Примечание

Яндекс.Музыка использует **виртуализацию списков** — треки рендерятся в DOM только при прокрутке. Поэтому нужно именно **прокручивать** плейлист, а не просто открыть его.
